Build **MiCloaker Lab Console**, a stable local Linux web app for acoustic/DAQ experiments. It is not an always-on production service: the user starts it before experiments, uses it through SSH port forwarding, and stops it after. Default bind must be `127.0.0.1`. Stability and simplicity are top priorities.

Use a structured Python project. Prefer **FastAPI + Jinja2 + vanilla JS/CSS**, no frontend build step. **Do not use any database.** Persist all state as plain text files under the workspace: JSON for per-session/per-run metadata, JSONL for append-only indexes/logs, CSV for result tables, Markdown for reports, and plain `.log` files. On restart, rebuild lists by reading these files. Provide mock DAQ mode and tests that pass without DAQ hardware. Import `uldaq` lazily only inside DAQ functions.

Core Linux workflow: create/open session → record run → save raw `.bin` float64 voltage → convert to WAV with scale tags → preview audio/plots → finalize metrics from saved `.bin` → compare `uj0` vs `uj1` → export files/ZIP. Raw `.bin` is the primary quantitative source. Peak WAV is listening-only. Range WAV is cross-check. WAV names must include `__scale-peak.wav` or `__scale-range-fs10V.wav`.

Required v0.1 features: session/run manager, metadata forms, recording job with text logs, bin→wav conversion, file browser, audio player, waveform/PSD/spectrogram plots, final RMS/Welch PSD/300–3400 Hz band power/dominant-tone metrics, uj0/uj1 attenuation dB, CSV/JSON/PNG/SVG results, individual/run/session/multi-session ZIP downloads, debug/log console with tracebacks.

Required v0.2 features: Live Monitor Mode with real-time waveform, RMS/peak meter, clipping warning, live PSD, scrolling spectrogram. Live values are preview-only. After recording ends, reload the saved `.bin`, recompute final metrics/plots/WAVs, update UI/metadata, and label finalized metrics as report-grade.

Add an **optional macOS Audio Helper** companion service for Mac playback control over Tailscale. It must not be required for Linux-only use. Implement manual Helper URL connection first; Tailscale discovery is optional. Linux must keep working if Helper is disconnected. Helper APIs: `/health`, `/devices`, `/files`, `/validate-playback`, `/play`, `/stop`, `/status`. It must play only WAVs under configured `wav_root`, choose output device by `device_id` without changing the system default output, validate file/device/sample-rate/channels before playing, and return clear errors. Store Helper status and playback info in run JSON/log files.

Keep the design modular but minimal. Avoid heavy frontend frameworks, databases, distributed sync, or features that reduce recording reliability. Provide `README.md`, `requirements.txt`, `requirements-mac-helper.txt`, tests, and clear run commands.
